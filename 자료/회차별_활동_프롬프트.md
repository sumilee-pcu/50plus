# 회차별 활동 전 추천 프롬프트 (전체 과정)

[목차로 돌아가기](../README.md)

각 회차 실습을 시작하기 전에 한 번 돌려 보면 좋은 프롬프트입니다. 수강생이 그대로 복사해 `[ ]` 안만 자기 것으로 바꾸면 됩니다.

**설계 원칙** — 모든 프롬프트는 같은 뼈대를 따릅니다: 역할(전문가) · 맥락(내 상황) · 지시(할 일) · 형식(결과 모양) · 검증(확실하지 않은 것은 표시). 근거는 [이론 근거](이론_근거.md)에 정리해 두었습니다.

> 공통 안전장치: AI는 모르는 것도 그럴듯하게 지어냅니다(할루시네이션). 그래서 프롬프트마다 숫자·출처·전망을 다시 확인할 장치를 넣었습니다.

---

## 1회차 · AI 트렌드와 업무혁신 (7/6, 이정기)

원리: 역할 부여 + 기회/위험/역량으로 구조화 + 불확실한 전망은 예측으로 표시(할루시네이션 안전장치).

```
당신은 AI 교육 컨설턴트입니다.
나는 [현재 직업 또는 강의 분야]에서 활동하려는 예비 AI 강사입니다.
2025~2030 AI 트렌드 중 내 업무나 강의에 직접 영향을 줄 변화 5가지를 뽑아,
각각을 기회 / 위험 / 준비할 역량 세 칸의 표로 정리해 주세요.
확실하지 않은 전망은 예측이라고 표시해 주세요.
```

---

## 2회차 · 프롬프트 엔지니어링과 AI 윤리 (7/8, 이수미)

원리: 프롬프트 자기 진단. 이 과정의 5대 원칙(역할·맥락·조건·예시·출력형식)으로 점검하고, 스스로 검증하게 합니다(Self-Verification 계열). 개선안을 2개로 받는 것은 여러 갈래를 비교하는 감각입니다.

```
아래 내가 쓴 프롬프트를 역할·맥락·지시(조건)·예시·출력형식 다섯 기준으로 진단해 주세요.
부족한 요소를 표로 지적하고, 더 좋은 프롬프트를 서로 다른 2개 버전으로 개선해 주세요.
개인정보·저작권·허위정보 위험도 함께 점검해 주세요.
프롬프트: [붙여넣기]
```

참고: 이 회차에서 다루는 Zero/Few-shot, CoT, ReAct, Reflexion, Self-Verification은 모두 실제 연구 기법입니다(few-shot: Brown 2020, CoT: Wei 2022·Kojima 2022, ReAct: Yao 2022, Reflexion: Shinn 2023). 오늘 실습의 4기법은 이 가운데 입문자에게 필요한 부분을 뽑은 것입니다.

---

## 3회차 · 이미지·콘텐츠·동영상 생성 (7/10, 권혁용)

원리: 역할 부여 + 이미지 프롬프트 공식(주제+스타일+구도+분위기)을 장면마다 적용 + 저작권 점검.

```
당신은 교육 콘텐츠 기획자이자 이미지 프롬프트 디렉터입니다.
[강의 또는 홍보 주제]를 위한 카드뉴스 5장 구성을 제안해 주세요.
각 장마다 핵심 문구 · 이미지 장면 · 이미지 생성 프롬프트 · 저작권 주의사항을 표로 정리해 주세요.
이미지 프롬프트에는 주제·스타일·구도·분위기가 모두 들어가게 해 주세요.
```

## 4회차 · 쇼츠·Long 영상 제작 (7/13, 권혁용)

원리: 3막 구조라는 내러티브 틀을 주어 결과를 좁힘 + 장면별 표로 형식 지정.

```
당신은 교육 홍보 영상 PD입니다.
[홍보할 강의 또는 나의 전문 분야]를 소개하는 30초 쇼츠 영상을 기획해 주세요.
3막 구조(도입-전개-마무리)로 나누고, 장면별로 메시지 · 화면 구성 · 자막 ·
BGM 분위기 · 촬영 또는 AI 생성 소스 준비물을 표로 정리해 주세요.
```

## 5회차 · 데이터 수집·분석·시각화 (7/15, 이애본)

원리: 큰 일을 단계로 분해(목적 → 데이터 → 수집 → EDA → 시각화) + 출처는 내가 재확인하도록 검색 키워드 요청.

```
당신은 데이터 분석 코치입니다. 나는 [분석하고 싶은 주제]를 분석하려 합니다.
분석 목적을 먼저 명확히 정리하고, 필요한 데이터 항목,
가능한 공공데이터 또는 API 출처 후보, 수집 후 EDA 절차, 추천 시각화 그래프를
단계별로 제안해 주세요.
출처는 내가 직접 다시 확인할 수 있게 검색 키워드도 함께 알려 주세요.
```

## 6회차 · AI 마케팅·홍보 전략 (7/20, 이애본)

원리: 같은 내용을 대상(Audience)에 따라 변형 + 민간형과 공공형의 목적 차이를 표로 대비.

```
당신은 공공기관 홍보와 민간 마케팅을 모두 이해하는 AI 전략가입니다.
[홍보 대상 또는 서비스]에 대해 민간형 메시지와 공공형 안내문을 각각 작성해 주세요.
대상은 50대 시민 · 실무자 · 의사결정자 세 버전으로 나누고,
톤 · 목표 · 행동 유도(CTA)의 차이를 표로 설명해 주세요.
```

## 6회차 · AI 에이전트와 업무 자동화 (7/20, 이애본)

원리: 업무 나누기(Divide Labor) + 사람이 확인할 지점과 오류 검증을 명시(자동화의 안전장치).

```
당신은 업무 자동화 설계자입니다. 내가 반복하는 업무는 [업무 설명]입니다.
이 업무를 AI 비서가 돕는 워크플로우로 설계해 주세요.
입력 자료 · 처리 단계 · 사람이 확인할 지점 · 최종 산출물 ·
오류가 생겼을 때 검증하는 방법을 표로 정리해 주세요.
```

## 7회차 · 교육 제안서 작성 (7/22, 양진욱)

원리: 제안서 목차라는 형식을 지정 + 역할 반전(평가위원 관점)으로 스스로 약점을 찾게 함.

```
당신은 공공기관 교육 제안서 컨설턴트입니다. [교육 주제] 과정 제안서를 만들려 합니다.
교육 필요성 · 대상 분석 · 학습목표 · 차시 구성 · 차별점 · 기대효과 · 평가 방법을
제안서 목차 형식으로 작성해 주세요.
그다음 평가위원 입장에서 중요하게 볼 근거와, 보완이 필요한 질문도 함께 제시해 주세요.
```

## 7회차 · AI 강사 브랜딩·포트폴리오 (7/22, 양진욱)

원리: 맥락(내 경력)을 재료로 + 과장 배제와 증거 필요 항목 표시(정직성·할루시네이션 안전장치).

```
당신은 강사 브랜딩 코치입니다.
내 경력은 [경력], 강의하고 싶은 분야는 [분야]입니다.
AI 강사로서의 전문 분야 · 강점 · 대표 강의 주제 · 포트폴리오 구성 · 1년 성장 계획을
표로 정리해 주세요.
과장된 표현은 빼고, 실제 증거가 필요한 항목은 표시해 주세요.
```

## 8회차 · 강의 설계·교안 작성 (7/27, 이애본)

원리: 교수학습지도안이라는 형식 지정 + 도입-설명-실습-평가 단계 분해.

```
당신은 교수설계 전문가입니다.
[대상]을 위한 [주제] 강의를 [시간]분으로 설계해 주세요.
학습목표 · 도입 활동 · 핵심 설명 · 실습 · 참여 도구(Quizizz·Kahoot·Mentimeter·Padlet 중 선택) ·
마무리 평가를 교수학습지도안 표로 작성해 주세요.
```

## 9회차 · 강의 시연·평가 (7/29, 이정기)

원리: 평가 항목(루브릭)이라는 형식 + 강점·개선점·바로 고칠 문장까지 구체적 피드백 요청(품질 평가).

```
당신은 AI 강사 시연 평가자입니다.
아래 강의 시연 계획을 도입 · 설명 명확성 · 실습 설계 · 시간 운영 ·
AI 활용 적절성 · 윤리와 저작권 안내 · 마무리 평가 항목별로 평가해 주세요.
강점 3가지, 개선점 3가지, 바로 고칠 문장 예시를 제시해 주세요.
계획: [붙여넣기]
```

## 생성형 AI는 어떻게 대답을 만드나

생성형 AI는 방대한 글을 학습해, 질문 뒤에 이어질 가장 그럴듯한 말을 확률로 계산합니다. 그래서 질문이 구체적일수록 답도 구체적으로 좁혀집니다. 모호하게 물으면 평균적인 답이 나옵니다.

```
내가 쓴 프롬프트  →  AI의 확률 계산  →  답변
(질문·조건·자료)     (이어질 말 예측)    (프롬프트의 질을 반영)
```

AI의 답은 프롬프트의 질을 넘지 못합니다.

---

## 프롬프트 비교

아쉬운 예

```
재취업 자기소개서 써줘
```

누구에게나 통하는, 나와 상관없는 평범한 글이 나옵니다.

좋은 예

```
당신은 중장년 재취업 컨설턴트입니다.
나는 25년간 백화점에서 고객 상담을 해왔고, 시니어 교육 강사에 지원합니다.
이 경력이 강사 직무에 왜 강점인지 중심으로,
자기소개서 초안을 600자 이내로 써 주세요.
```

내 경력이 담긴, 바로 다듬어 쓸 수 있는 초안이 나옵니다. 차이는 누가, 왜, 무엇을, 어떻게를 담았는가 하나입니다.

---

## 좋은 프롬프트의 네 요소

| 요소 | 뜻 | 예시 |
|---|---|---|
| 역할 (Role) | AI에게 전문가 역할을 입힌다 | 당신은 중장년 취업 컨설턴트입니다 |
| 맥락 (Context) | 내 상황과 배경을 알려준다 | 나는 25년 경력의 유통 관리자입니다 |
| 지시 (Instruction) | 원하는 작업을 명확히 시킨다 | 강점 세 가지를 뽑아 정리해 주세요 |
| 형식 (Format) | 결과물의 모양을 지정한다 | 표로, 600자 이내로, 존댓말로 |

---

## 기본 템플릿

```
당신은 [ 분야 ]의 전문가입니다.          ← 역할

나는 [ 나의 상황 · 경력 ]입니다.          ← 맥락

[ 원하는 작업 ]을 해 주세요.             ← 지시

결과는 [ 분량 · 형식 · 어조 ]로 정리해 주세요.  ← 형식
```

네 요소를 다 채우지 못해도 됩니다. 역할과 맥락만 넣어도 결과가 크게 달라집니다. 자주 쓰는 프롬프트는 메모장에 저장해 두고 다시 씁니다.
